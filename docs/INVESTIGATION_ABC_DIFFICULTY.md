# Pair-link 難易度「振れ幅」調査レポート

## 調査目的

1. Cスコア（分岐の複雑性）にボラティリティを持たせることが可能か
2. A（迂回）および B（囲い込み）を Level 1〜10 まで意図的に操作できるか
3. 現行アルゴリズムの構造的制約の特定

---

## 1. Cスコアが 1.611 で固定される原因

### 数学的証明

**Cスコアは「全マスを埋める」制約下で、6×6・5ペアの幾何学的必然により完全に固定される。**

- 6×6 グリッドの辺数: 60（水平30 + 垂直30）
- 各パスは単純鎖（分岐なし）。パス長 L のパスは (L-1) 本の同パス辺を持つ
- 全36セルを5パスで分割 → 同パス辺の総数 = Σ(L_i - 1) = 36 - 5 = **31**
- **異パス間の辺数 = 60 - 31 = 29**（常に一定）

Cスコアの定義: 各パスセルにおける「他パス隣接セル数」の平均。  
各異パス辺は両端のセルからそれぞれ1回ずつカウントされるため、合計 = 29 × 2 = 58。

```
C = 58 / 36 ≈ 1.611（不変）
```

**結論**: 枝刈りや評価関数とは無関係。グリッドサイズ・ペア数・全マス埋めの条件下で、Cはトポロジーにより一意に決まる。

### コード上の確認

- `evaluateForestGrid`: filled×10 と emptyIsolatedPenalty のみ。ABCは未考慮
- `generateFullCoverByBeam`: 全マス埋めを前提にビーム探索
- Cスコアに変動を持たせるには、**全マス埋めを緩める**か、**Cの定義を変更**する必要がある

## 3. 難易度制御の可否

### A（迂回率）

- **現状**: 1.25〜3.40 の範囲で自然に変動
- **Level 1 (A < 1.2)**: 100回中0回 → 現行ロジックでは事実上不可
- **制御可能性**: `evaluateForestGrid` に迂回ペナルティを追加し、直線に近い経路を優先すれば、理論上は可能

### B（エンクロージャ）

- **現状**: 2〜11 の範囲で変動
- **Level 10 (B > 10)**: B max=11 のため、ごく稀に達成の可能性
- **制御可能性**: 評価関数で囲い込みを明示的に加味すれば、意図的な増減は可能

### C（分岐の複雑性）

- **現状**: 1.611 で完全固定
- **Level 10 (C > 2.0)**: 6×6・全マス埋めでは**幾何学的に不可能**
- **制御可能性**: 現定義のままでは不可。定義変更（例: 分岐点数・袋小路数など）が必要

---

## 4. Cスコアにボラティリティを持たせるための提案

### 案A: 全マス埋めの緩和

- 一部セルを「未使用」として残し、袋小路を許容する
- 現行の一意解チェック・ビーム探索の大幅改修が必要
- Cの変動は可能になるが、パズルとしての性質が変わる

### 案B: Cの再定義

- 「隣接他パス数」ではなく、例えば以下を採用:
  - **分岐点数**: パス上の「3方向以上が空き」のセル数
  - **袋小路密度**: 正解ルートから外れた選択肢の数
- 全マス埋めを維持したまま、別の難易度指標として使う

### 案C: グリッドサイズの活用

- 8×8, 10×10 では C の値が異なる（トポロジーが変わる）
- サイズごとに C の理論値が変わるため、難易度の段階付けに利用可能

## 5. 境界線の整理

| 設定 | 現行ロジックでの可否 |
|------|----------------------|
| A < 1.2, B = 0 | **不可**（100回中0回） |
| B > 10 | **稀に可能**（B max=11） |
| C > 2.0 | **幾何学的に不可能** |
| A 1.5〜2.5, B 4〜8 | **自然に達成**（現状の主力帯） |
